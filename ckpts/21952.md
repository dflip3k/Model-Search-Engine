# Eris

#### Civit ID: 21952

<p><strong><em>Eris</em></strong><em>, the Greek goddess of strife and discord.</em></p><p></p><p>This is my first pure 3D render/Cartoon model. When I started working on this model I had almost zero knowledge on writing prompts for 3d renders/CGI. As such, my main goal for this model was quickly realized - Make a model that accepts photorealistic prompts and returns beautiful 3D rendered outputs. Normal 3D render/CGI prompts also work with this model.</p><p></p><p></p><h1 id="heading-21"><strong>Important Settings</strong></h1><p>The following settings assume you're using webui by Automatic1111</p><ul><li><p><strong><u>USE CFG VALUES BELLOW 8*</u></strong></p><ul><li><p>This model does not respond well to CFG values above 8. With some caveats that will be explained bellow.</p></li><li><p><strong>Do not</strong> go over <strong>6 CFG</strong> with <strong>Euler a</strong> unless,</p></li><li><p>You are using Hires Fix. I found that you can go as high as 10 CFG with hires. fix enabled. <strong>This applies to all samplers</strong>.</p></li><li><p><strong>DPM++ SDE Karras</strong> starts to degrade around<strong> 7 CFG</strong> if using <code>clip skip = 1</code>, no hires fix. <strong>However</strong>, if you're using <strong>DPM++ SDE Karras</strong> with <code>clip skip = 2</code> the CFG values can go up to at least <strong>10 CFG</strong> without significant (if any) degradation.</p></li><li><p>General Advice:</p><ul><li><p><strong>Do Not</strong> exceed <strong>6 CFG</strong> with <strong>Euler a</strong>, <strong>unless using hires fix</strong>.</p></li><li><p><strong>Do Not</strong> exceed <strong>7 CFG</strong> with <strong>DPM++ SDE Karras</strong> with <strong>clip skip set to 1</strong>, <strong>unless using hires fix</strong>.</p></li><li><p><strong>You Can</strong> go up to <strong>at least 10 CFG</strong> with <strong>DPM++ SDE Karras</strong> with <strong>clip skip set to 2</strong>, <strong>without using hires fix</strong>.</p></li><li><p>I could not test all samplers. <strong>If you notice broken compositions or distortions</strong>; <strong>lower </strong>your <strong>CFG </strong>value <em>and/or</em> <strong>enable hires fix</strong>. My own personal settings for hires fix can be found bellow.</p></li></ul></li><li><p>Thanks to Duo V2 on the Unstable Diffusion discord for helping me get to the bottom of this issue.</p></li></ul></li><li><p>VAE: vae-ft-mse-840000-ema-pruned.safetensors</p><ul><li><p>link - <a target="_blank" rel="ugc" href="https://huggingface.co/stabilityai/sd-vae-ft-mse-original/blob/main/vae-ft-mse-840000-ema-pruned.safetensors">https://huggingface.co/stabilityai/sd-vae-ft-mse-original/blob/main/vae-ft-mse-840000-ema-pruned.safetensors</a></p></li><li><p>This is already baked into the model but it never hurts to have VAE installed.</p></li><li><p>Under <code>Settings -&gt; Stable Diffusion -&gt; SD VAE -&gt;</code> select the VAE you installed via dropdown. Then, uncheck <code>Ignore selected VAE for stable diffusion checkpoints that have their own .vae.pit next to them</code></p></li></ul></li><li><p><strong>Clip skip : 2</strong></p><ul><li><p>This varies. Most of the time <code>Clip skip = 2</code> works best with this model. In the event you're not getting the results you want. First, try <code>Clip skip = 1</code>.</p></li></ul></li><li><p>ETA Noise Seed Delta: 31337</p><ul><li><p>Found in - <code>Settings -&gt; Sampler parameters -&gt; Eta noise seed delta</code></p></li></ul></li><li><p>Sigma Noise: 1</p></li><li><p>Sigma Churn: 0 <strong>or</strong> 1</p><ul><li><p><strong>Optional</strong> - When using Euler, Heun, DPM2 samplers try setting <code>Sigma Churn = 1</code></p></li><li><p>Found in - <code>Settings -&gt; Sampler parameters -&gt; sigma churn</code></p></li></ul></li><li><p><strong>Optional</strong> Negative Textual Inversion (TI): bad_prompt_version2</p><ul><li><p>link - <a target="_blank" rel="ugc" href="https://huggingface.co/datasets/Nerfgun3/bad_prompt/blob/main/bad_prompt_version2.pt">https://huggingface.co/datasets/Nerfgun3/bad_prompt/blob/main/bad_prompt_version2.pt</a></p></li><li><p>Install to - <code>stable-diffusion-webui -&gt; embeddings</code></p></li><li><p>In webui - Add <code>(bad_prompt_version2:0.8)</code> at the end of your negative prompt.</p></li></ul></li><li><p><strong>Optional</strong> Recommended Samplers:</p><ul><li><p>DPM++ SDE Karras // 18-35 steps</p></li><li><p>Euler // 40 - 80 steps // Sigma Churn : 1</p></li><li><p>DPM++ 2M Karras // 40 - 70 steps</p></li><li><p>DPM++ 2S a Karras // 30 - 75 steps</p></li><li><p>Heun // 20 - 30 steps // Sigma Churn : 1</p></li><li><p>DPM2 // 30 - 75 steps // Sigma Churn : 1</p></li><li><p>These are just recommendations; feel free to experiment with different values and samplers.</p></li></ul></li><li><p><strong>Optional</strong> Hires. Fix settings (these are my go-to parameters):</p><ul><li><p>Upscaler : R-ESRGAN 4x+</p></li><li><p>Denoising strength: 0.3 - 0.35</p></li><li><p>The rest of the settings are up to you. They don't really impact the quality of the output.</p></li><li><p>Note: Hires Steps, if you're using high steps for SD - You can choose a hires step value lower than your SD step value. This will increase the generation times.</p><ul><li><p>For example; Let's say you're using Euler sampler @ 60 steps. You can then set the Hires Steps to ~40. Apposed to the default which is 0. Hires Steps = 0 means it will follow the same step value as sampler (Euler in this case). Lower Hires Step values will reduce generation time.</p></li></ul></li></ul></li><li><p><strong>Optional</strong> Face Restoration:</p><ul><li><p>I use CODEFORMER @ 0.8 strength. Whether or not you use face restoration is up to you.</p></li></ul></li><li><p><strong>Optional</strong> Quick Settings:</p><ul><li><p>For easier access to some of the settings in webui, you can move some settings sliders/dropdowns to the main interface (beside model selection).</p></li><li><p>Quick settings location -</p><ul><li><p>Settings -&gt; User interface -&gt; Quicksettings list</p></li></ul></li><li><p>My Quick settings -</p><ul><li><p><code>sd_model_checkpoint, CLIP_stop_at_last_layers, s_churn</code></p></li><li><p>Where:</p><ul><li><p><code>CLIP_stop_at_last_layers</code> is Clip Skip</p></li><li><p><code>s_churn</code> is sigma churn</p></li></ul></li></ul></li></ul></li></ul><p></p><p><u>PDF documentation is in the works.</u></p><p></p><p><strong><u>Check out my other models</u></strong></p><ul><li><p>Based Model -
